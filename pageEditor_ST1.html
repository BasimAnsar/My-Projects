Splash Page

Technical Description

The Splash page was designed to create an engaging and informative introduction to the website. I used HTML5 for structure and CSS3 for styling and animations. A key technique was employing a `div` with a background image (`splash-screen` class) to set the overall visual tone. The falling letter animation was achieved using CSS keyframes (`@keyframes fall`) applied to individual `div` elements (`letter-box` class) containing each letter. The `setTimeout` function in Javascript was used to automatically redirect to the main page after 5 seconds.

The main structure consists of a `div` with the `splash-screen` class which holds the main content within it. Inside, the `content` div displays a title and a brief paragraph. The animated letters use nested `div` elements (`letter-container`, `letter-row`, `letter-box`) to arrange the characters and apply individual delays to create the falling effect. Finally, a `loader` div contains the spinner animation created using CSS `border` and `animation` properties.

Key CSS properties used were: `background-image`, `background-size`, `background-position` to style the background; `animation` and `@keyframes` to animate the falling letters and the loader; `display: flex` and `justify-content` to center elements; and `setTimeout` with Javascript to redirect the user.

Accessibility

For accessibility, I ensured that the background image had sufficient contrast with the text. The animated elements do not convey essential information, so they do not hinder accessibility. Alt text would be provided for the logo if one was being used. The timed redirect is kept short to minimize the time a user might be disoriented.

Volunteer Page

Technical Description

The Volunteer page provides a visually appealing and interactive interface for browsing volunteer opportunities. I used HTML5 for the structure and CSS3 for styling, including flexbox and grid layouts for responsiveness. Each volunteer opportunity is presented in a card format (`volunteer-card` class) that includes an image, title, location, rating, tags, a brief description, and a "Details" button.

The page utilizes a grid layout for arranging the volunteer cards and flexbox for the card content. A dropdown menu (`filter-dropdown`) allows users to sort opportunities. The cards include images (`img` tag), headings (`h3` tag), paragraph descriptions (`p` tag), and interactive button. CSS is used to style the card layout, images, and hover effects for enhanced user experience.

Key CSS properties used are: `display: grid` and `grid-template-columns` for the volunteer card layout; `display: flex` to arrange the card content; `object-fit: cover` for image sizing; and `transition` for the hover effects. The feedback section uses CSS to create a responsive form with styled textarea and buttons.

Accessibility

Accessibility considerations include ensuring sufficient contrast between text and background colors, providing alt text for all images, and using semantic HTML to structure the content. The "Details" buttons should link to more descriptive pages or pop-ups. The sort dropdown is designed to be keyboard accessible. ARIA attributes could be added to provide better screen reader support.

Content Page

Technical Description

The Content page is designed to present information about quality education in a structured and accessible manner. HTML5 is used for semantic structure and CSS3 for styling. An internal navigation menu (`internalNav`) allows users to quickly jump to different sections of the page. Each section (`section` tag) includes a heading, descriptive text, and relevant images.

The `internalNav` is created using a `nav` element with an unordered list (`ul`) of anchor links (`a` tag). The `section` elements are styled with a white background and box shadow for visual separation. The images are set to `max-width: 100%` to ensure responsiveness. A "Back to Top" link provides easy navigation to the top of the page.

Key CSS properties used are: `position: sticky` to keep the internal navigation menu visible; `display: block` for consistent link styling; and `max-width: 100%` for responsive images. The javascript is used to add back to top functionality.

Accessibility

Accessibility features include using semantic HTML5 tags (e.g., `header`, `nav`, `main`, `section`, `footer`), ensuring sufficient contrast between text and background colors, and providing descriptive alt text for all images. The internal navigation menu helps users quickly navigate the page. ARIA attributes and keyboard navigation should be further tested to improve accessibility.

Challenges and Lessons Learned

One challenge was creating the falling letter animation for the Splash page, ensuring it was both visually appealing and performed well across different browsers. I learned to fine-tune the CSS keyframes and delays to achieve the desired effect without impacting performance. Another challenge was creating responsive layouts for the Volunteer page, particularly the volunteer card grid. I learned to use CSS grid and media queries effectively to adapt the layout to different screen sizes. Additionally, I underestimated the importance of thorough cross-browser testing and accessibility considerations early in the development process. I learned to prioritize these aspects to ensure a consistent user experience and compliance with web standards.

Compliance

The pages I created aim to be compliant with Janet regulations, focusing on accessibility, privacy, and data protection. Accessibility is addressed by providing alt text for images, using semantic HTML, and ensuring sufficient color contrast. To comply with privacy regulations, no personal data is collected or stored on these static pages. I am committed to following best practices for secure coding and data handling as required by Janet regulations. However, a full compliance audit is necessary to ensure complete adherence to all applicable standards.
