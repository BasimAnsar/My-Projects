feedback Page

Technical Description

1. HTML Structure:

  • Navigation Bar: The navigation bar (nav) contains links (<a>) to various pages of the website, making it easy to navigate between sections like Home, Volunteer, Profile, etc. The alt attribute for the logo image (<img>) ensures that users with visual impairments can understand the content of the image via screen readers.
  • Team Section: The team section includes a main title (<h1>) and a subtitle (<h2>) describing the purpose of the page, followed by a grid layout to showcase team members. Each team member is represented in a card-style format (<div> with class card-set), which includes the member's image, name, role, and responsibilities.
  • Team Member Cards: Each team member is presented in a card-set with a profile picture (<img>) and a list of tasks (<ul>). The cards are styled to be visually interactive, using hover effects to enhance user engagement.

2. CSS Styling:

  • Grid Layout: The team-grid class uses CSS Grid to arrange the team cards in a responsive layout. The cards adjust based on the screen size, ensuring that they remain organized and readable on different devices.
  • Hover Effects: Each card-set has a hover effect that enlarges the profile image and elevates the card, giving a dynamic interaction to the user. The list of duties (tasks) is also animated to show or hide when the card is hovered.
  • Typography and Colors: The page uses a green color palette (#228b22) for text, titles, and button elements to maintain the environmental theme. The font family is set to 'Poppins' to provide a modern and clean look.

3. Accessibility Considerations:

  • Alt Text for Images: All images in the team section have descriptive alt attributes, such as alt="Nanduni Pathirana, Frontend Developer | Student 1", to provide meaningful descriptions for screen readers.
  • Keyboard Navigation: The tabindex="0" attribute is added to the card-set elements to make them focusable, ensuring that users can navigate through the team cards using the keyboard.
  • Focus Styles: The :focus pseudo-class is applied to card-set elements, giving them a visible outline when they are focused, improving accessibility for keyboard and assistive technology users.
  • Semantic HTML: The page uses semantic HTML tags like <header>, <main>, and <footer> to ensure that the page structure is meaningful and easy to navigate with screen readers.
  • Responsive Design: The layout is responsive, adapting to smaller screen sizes using media queries. For instance, on smaller devices (max-width: 1024px), the team grid switches to a two-column layout, and on mobile (max-width: 480px), the layout switches to a single-column layout for better readability.

Content Page

Technical Description

1. HTML Structure:

  • Navigation Bar: The navigation bar is used to access different pages of the website, such as Home, Volunteer, and Statistics. It provides easy navigation with clearly labeled links. The logo image (<img>) has an alt attribute for screen readers to describe the image content, ensuring accessibility for visually impaired users.
  • Sidebar Navigation: The sidebar contains a vertical navigation menu (<nav>) with links to different sections of the page, allowing users to easily jump to specific content. This improves accessibility and user experience, particularly for users with mobility impairments.
  • Content Sections: The content is organized into sections with <section> tags, each with a clear heading (<h2>) to guide users through the text. This is especially helpful for screen readers to organize the content and navigate the sections.
  • Image Placeholders: Each section includes images to visually support the content. These images are contained within <div> elements with the image-placeholder class, ensuring that the layout remains responsive.

2. CSS Styling:

  • Responsive Layout: The page uses a responsive layout, with content sections displayed in a two-column layout when viewed on larger screens and a single-column layout on smaller screens. This is achieved using media queries in the CSS, ensuring that the content is readable on all devices.
  • Hover Effects: The sidebar navigation links have a hover effect to provide visual feedback to users when interacting with the menu. This enhances user engagement and ensures a dynamic experience.
  • Color Scheme: The page uses green and neutral colors to align with the environmental theme, ensuring that text and background colors have a high contrast for readability. These choices also provide a clean and modern look.

3. Accessibility Considerations:

  • Alt Text for Images: All images used on the page, such as in the introduction and conclusion sections, have descriptive alt attributes. This is vital for users who rely on screen readers to understand the content of the images.
  • Semantic HTML: The page uses semantic HTML tags like <header>, <nav>, and <footer> to ensure proper structure. This improves accessibility by making the content more understandable to assistive technologies.
  • Navigation: The page is designed to be easily navigable via the keyboard, with clear tabbing order, especially within the sidebar and content sections. Users can use the tab key to navigate between links and form elements.
  • Focus Styles: The :focus pseudo-class is used to indicate the active element, making it easier for users with keyboard navigation to identify where they are on the page. This is especially useful for screen reader users and those who rely on keyboard navigation.
  • Forms and Buttons: The form elements, such as the text inputs and radio buttons, are properly labeled using <label> tags. This ensures that screen readers can properly announce each form element. Additionally, buttons have clear labels and a hover effect for better interaction.

Challenges and Lessons Learned

Challenges:

  • Responsiveness: Implemented CSS media queries to ensure compatibility across devices.
  • Navigation Accessibility: Used semantic HTML and ARIA attributes for screen reader support.
  • Hover/Focus Effects: Ensured interactive elements were accessible via keyboard and screen readers.
  • Image Accessibility: Added alt attributes to all images for better accessibility.

Lessons Learned:

  • Prioritize Accessibility: Accessibility should be prioritized early in development.
  • User-Centered Design: Ensured responsive design for all devices.
  • Performance: Optimized multimedia content for faster load times.
  • CSS Animations: Used simple animations for user engagement without impacting performance.
  • Cross-Browser Compatibility: Ensured consistent performance across all browsers.
  • Iterative Design: Regular testing and iteration improved usability.

Compliance

Our web pages have been designed with full compliance with Janet regulations to ensure accessibility, data security, and best practices in web publication.

Key Compliance Areas:

  • Accessibility: All images have descriptive alt attributes for screen reader compatibility. We use semantic HTML tags for better navigation, ensuring accessibility for users with disabilities.
  • Security: Data collection forms are secure, and sensitive information is protected through HTTPS. We implement security measures to protect users' privacy and data.
  • Content Accuracy: Content is regularly updated to reflect accurate and truthful information, adhering to guidelines for truthful online communication.
  • Data Handling: We follow data protection policies, collecting only the necessary data from users and ensuring that their privacy is maintained in accordance with applicable regulations.
  • Responsive Design: We ensure our pages are fully responsive and accessible across all devices, complying with Janet’s requirements for device compatibility.

By following these guidelines, we ensure that our website is compliant with Janet's standards and is accessible to all users, regardless of their abilities or devices.
